FASE 4 - Superadmin sin exponer contrasenas (sin romper gestion)
Fecha: 2026-03-05

Objetivo
- Evitar fuga de password en panel superadmin.
- Mantener el flujo de crear/editar medicos.

Archivos modificados
1) backend/API.gs.txt
2) js/superadmin.js
3) superadmin.html

CAMBIO 1 - API ya no devuelve password en superadmin_get_data
Archivo: backend/API.gs.txt (getSuperadminData_)

Antes:
- Se exigia columna password y se devolvia password por cada medico.

Ahora:
if (idxDName === -1 || idxDUser === -1) {
  return { success: false, message: "usuarios_admin requiere: usuario, nombre_doctor, rol, correo_notificaciones, telefono." };
}

Y en doctors.push(...) se retiro password:

doctors.push({
  nombre_doctor: ...,
  usuario: ...,
  rol: ...,
  correo_notificaciones: ...,
  telefono: ...
});

CAMBIO 2 - Editar medico sin enviar nueva clave conserva la actual
Archivo: backend/API.gs.txt (superadminUpdateDoctor_)

Validacion ajustada:
if (!oldUsuario || !newUsuario || !nombreDoctor) {
  return { success: false, message: "Faltan datos del medico." };
}

Actualizacion de password solo si viene valor:
if (password) {
  doctorsSheet.getRange(rowIndex, idxDPass + 1).setValue(password);
  if (idxFirst > -1) doctorsSheet.getRange(rowIndex, idxFirst + 1).setValue("SI");
}

CAMBIO 3 - Formulario superadmin no precarga contrasena
Archivo: js/superadmin.js

Antes:
document.getElementById("doctorPassword").value = doctor ? (doctor.password || "") : "";

Ahora:
document.getElementById("doctorPassword").value = "";

Validacion de guardado:
if (!payload.nombre_doctor || !payload.usuario || (mode === "create" && !payload.password)) {
  showMsg_(mode === "create"
    ? "Completa nombre, usuario y contrasena."
    : "Completa nombre y usuario.");
  return;
}

CAMBIO 4 - Campo password opcional al editar (UI)
Archivo: superadmin.html

<input id="doctorPassword" class="input-field" data-format-hint="Opcional al editar: si la dejas vacia, se mantiene la contrasena actual.">

Impacto
- El superadmin deja de ver contrasenas en texto plano de medicos.
- Crear medico sigue pidiendo contrasena.
- Editar medico mantiene contrasena actual si el campo queda vacio.
