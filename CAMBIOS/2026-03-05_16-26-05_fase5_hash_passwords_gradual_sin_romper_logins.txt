FASE 5 - Hash de contrasenas gradual (compatibilidad incluida)
Fecha: 2026-03-05

Objetivo
- Dejar de guardar nuevas contrasenas en texto plano.
- Mantener compatibilidad con usuarios ya existentes (texto plano heredado).

Archivo modificado
1) backend/API.gs.txt

CAMBIO 1 - Login compatible con hash + texto plano
Se reemplazo comparacion directa por verificacion segura:

if (superadmin && verifyPassword_(data.password, superadmin.password)) { ... }
if (admin && verifyPassword_(data.password, admin.password)) { ... }
if (paciente && verifyPassword_(data.password, paciente.password)) { ... }

CAMBIO 2 - Helpers nuevos de password
Se agregaron funciones:

function isPasswordHash_(value) { ... }
function hashPassword_(plainPassword) { ... }
function verifyPassword_(inputPassword, storedPassword) { ... }
function toStoredPassword_(value) { ... }
function preparePasswordForSheet_(sheetName, payload) { ... }

Detalle clave:
- Formato guardado: sha256$<salt>$<hash>
- Si password guardado no esta en hash, verifyPassword_ compara en modo legacy (texto plano).

CAMBIO 3 - create_record ahora prepara password segun hoja

En create_record se agrego:
data.data = preparePasswordForSheet_(data.sheet, data.data);

Y en rama general:
response = createRecord(
  data.sheet,
  preparePasswordForSheet_(data.sheet, normalizePayloadForSheet_(data.sheet, data.data || {}))
);

CAMBIO 4 - Escrituras de password migradas a hash
Se cambio a toStoredPassword_(...) en:
- completeFirstLogin_
- updateSelfPassword_
- superadminUpdateDoctor_ (solo si se envio nueva password)
- superadminCreateDoctor_
- superadminUpdatePatientPassword_
- superadminUpdatePatientManagement_
- seed de ensureDefaultSuperadmin_

Ejemplo aplicado:

sheet.getRange(rowIndex, idxPass + 1).setValue(toStoredPassword_(pass));

Impacto
- Nuevas contrasenas quedan hasheadas.
- Usuarios antiguos siguen entrando aun si su password historica estaba en texto plano.
- No se toca WhatsApp de notificaciones (puede seguir enviando la clave inicial ingresada en UI).
